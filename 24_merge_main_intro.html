Boas práticas 3/3 · Estratégias de merge na sua branch main

Como integrar na branch principal?

> Ao aceitar um PR, a plataforma costuma oferecer mais de uma forma de levar o trabalho para a main — cada opção desenha o git log --graph de um jeito diferente.
> Alinhe regra + convenção no time; nos próximos slides, três formatos típicos com antes → depois no desenho do histórico.

Squash

Vários commits do PR viram um só na main.

ANTES  PR #42 · 3 commits
    * E  wip
    * D  testes
    * C  scaffold
    /
main * B
   * A

DEPOIS  histórico “fino”
main * S  feat: entrega (#42)
       * B
       * A

Merge commit

Aparece o nó de merge ligando as linhas.

ANTES  duas pontas vivas
main   * B  ← produção
         |
feat * D
       * C

DEPOIS  nó M com 2 pais
*   M  Merge PR #42
|\
| * D
| * C
* / B
|/
* A

Rebase + merge

Commits reencaixados: linha reta na main.

ANTES  C e D “pendurados”
main   * B
         |
feat * D
       * C

DEPOIS  reaplicados · novos SHAs
main   * D′  mesmo patch,
         * C′  hash diferente
         * B
         * A
