Boas práticas 2/3 · PR com boa descrição e contexto

Boas práticas de Pull Request

> Com os commits organizados e o histórico mais legível no branch, chegamos à hora de pedir revisão — e é aí que entra o PR.
> O PR não é só “juntar código”: é onde você entrega contexto para quem vai ler o diff — o que mudou, por quê e como validar.
