Rebase interativo

> Rebase interativo (git rebase -i) reescreve o histórico local: você pode juntar commits (squash / fixup), reordenar, editar mensagens (reword) ou remover commits (drop) antes de abrir o PR.
> O Git abre um arquivo “todo” com uma linha por commit e verbos (pick, squash…). Salvar e fechar o editor continua o fluxo; em squash você costuma unificar a mensagem final na sequência.
> Use com prioridade em branch só sua ou ainda não mergeada. Se o branch já foi empurrado para o remoto, reescrever exige force-with-lease e alinhamento com o time — o ideal é limpar antes do push.
# Rebase interativo (ex.: últimos 3 commits)
git rebase -i HEAD~3

# A partir de um commit base (ancestral comum)
git rebase -i <hash-ou-main>

# Se precisar cancelar o rebase em andamento
git rebase --abort
