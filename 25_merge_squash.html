Boas práticas 3/3 · Estratégias de merge na sua branch main

Squash and merge

> O que é: o PR inteiro vira um único commit na ponta da main — você ajusta a mensagem final no botão de merge.
> Quando: WIP e muitos commits pequenos no branch; main fina e leitura rápida do que entrou por PR.
> Trade-off: na main some o detalhe commit a commit — bom título + descrição do PR (ou corpo do squash) carregam o contexto.

No branch (antes)

Cenário Três commits pequenos no PR; main parada em B.

* B  main ← ponta atual
|\
| * E  docs: typos
| * D  test(api): casos 404
| * C  feat(api): rota /busca
|/
* A  base comum

Na main (depois do squash)

Efeito C, D e E somem como nós próprios; vira S com mensagem que você edita no merge.

* S  feat(api): busca + testes + docs (#42)
|            ↑ 1 commit = 1 PR inteiro
* B  main ← continua linear
* A
